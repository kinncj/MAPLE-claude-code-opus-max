# docs/design/

Design artifacts live here. Structure is maintained by design agents; do not manually reorganise.

```
docs/design/
├── research/           # Personas, journey maps, research summaries (ux-researcher)
├── wireframes/         # Per-story wireframe files (wireframe-architect)
├── mockups/            # Per-story mockup code + audit reports (ui-mockup-builder, a11y-auditor)
├── identity/           # Visual identity: palette, typography, canonical tokens + framework emits
│   ├── palette.json        ← visual-identity-designer output
│   ├── typography.json     ← visual-identity-designer output
│   ├── tokens.json         ← W3C DTCG canonical (design-system-author)
│   ├── tokens.css          ← emitted CSS vars (design-tokens skill)
│   ├── tailwind.tokens.js  ← emitted Tailwind config (design-tokens skill)
│   └── mantine.theme.ts    ← emitted Mantine theme (design-tokens skill)
└── system/
    └── components/     # Component token inventory (design-system-author)
```

## Key Rules

- `identity/tokens.json` is the only file agents and humans edit. All framework emits are regenerated from it.
- Wireframes must be marked `status: approved` before mockup generation can proceed.
- Palette must be marked `status: approved` before token propagation.
- Mockups must be marked `status: approved` before component scaffold runs.
- Audit reports (`.a11y.json`) are generated by the `a11y-auditor` agent — do not commit empty stubs.
